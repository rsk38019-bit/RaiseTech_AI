# ユースケース

## アクター

- **ユーザー**：ブラウザからアプリを操作する人

---

## ユースケース一覧

| UC | ユースケース名 | 概要 |
|----|--------------|------|
| UC-01 | ボードを閲覧する | アプリを開くと全カラムと各カードが一覧表示される |
| UC-02 | カードを作成する | 任意のカラムにカードを新規追加する |
| UC-03 | カードを編集する | 既存カードのタイトルを変更する |
| UC-04 | カードを削除する | 不要なカードを削除する |
| UC-05 | カードを移動する | カードを隣のカラムへ移動し、進捗を更新する |

---

## ユースケース詳細

### UC-01 ボードを閲覧する

| 項目 | 内容 |
|------|------|
| アクター | ユーザー |
| 事前条件 | なし |
| 基本フロー | 1. ユーザーがアプリにアクセスする<br>2. フロントエンドがAPIからボード情報を取得する<br>3. カラムとカードが画面に表示される |
| 事後条件 | ボード・カラム・カードが画面に表示されている |
| 例外フロー | API取得に失敗した場合、エラーメッセージを表示する |

### UC-02 カードを作成する

| 項目 | 内容 |
|------|------|
| アクター | ユーザー |
| 事前条件 | ボードが表示されている |
| 基本フロー | 1. ユーザーがカラムの「+ カードを追加」をクリックする<br>2. インラインフォームが表示される<br>3. ユーザーがタイトルを入力し追加ボタンを押す<br>4. カードが作成されカラムに表示される |
| 事後条件 | 新しいカードがカラムに追加されている |
| 例外フロー | タイトルが空の場合、エラーメッセージを表示しカードは作成しない |

### UC-03 カードを編集する

| 項目 | 内容 |
|------|------|
| アクター | ユーザー |
| 事前条件 | 編集対象のカードが存在する |
| 基本フロー | 1. ユーザーがカードのタイトルをダブルクリックする<br>2. タイトルが編集可能な入力欄に変わる<br>3. ユーザーがタイトルを変更しEnterを押す<br>4. 変更内容が保存され表示に反映される |
| 事後条件 | カードのタイトルが更新されている |
| 例外フロー | 変更後のタイトルが空の場合、保存せず元のタイトルに戻す |

### UC-04 カードを削除する

| 項目 | 内容 |
|------|------|
| アクター | ユーザー |
| 事前条件 | 削除対象のカードが存在する |
| 基本フロー | 1. ユーザーがカードの削除ボタンをクリックする<br>2. 確認ダイアログが表示される<br>3. ユーザーが「削除」を選択する<br>4. カードが削除され一覧から消える |
| 事後条件 | カードが削除されている |
| 例外フロー | 確認ダイアログで「キャンセル」を選択した場合、カードは削除されない |

### UC-05 カードを移動する

| 項目 | 内容 |
|------|------|
| アクター | ユーザー |
| 事前条件 | 移動対象のカードが存在する |
| 基本フロー | 1. ユーザーがカードをドラッグする<br>2. ドロップ先カラムがハイライトされ、挿入位置がインジケーターで示される<br>3. ユーザーが目的のカラムの任意の位置にドロップする<br>4. カードが指定位置に挿入され、移動元・移動先のカード枚数が更新される |
| 事後条件 | カードが指定したカラムの指定位置に移動している |
| 例外フロー | なし |
