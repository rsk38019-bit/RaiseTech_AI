# 要件定義書 — Trello風タスク管理アプリ

**バージョン:** 1.0  
**作成日:** 2026-05-02  
**作成者:** rsk38019-bit

---

## 1. プロジェクト概要

### 1.1 目的

シンプルなカンバンボード形式のタスク管理アプリを提供する。ユーザーがタスクをカード単位で管理し、進捗状況をカラム間の移動で可視化できるようにする。

### 1.2 スコープ

- シングルページアプリケーション（SPA）
- ローカル状態のみ（バックエンド・DB なし）
- ブラウザのローカルストレージによるデータ永続化

## 3. 機能要件

### 3.1 ボード

| ID | 要件 |
|----|------|
| F-01 | アプリ起動時に単一のボードを表示する |
| F-02 | ボードはデフォルトで「To Do」「In Progress」「Done」の3カラムを持つ |

### 3.2 カラム

| ID | 要件 |
|----|------|
| F-03 | 各カラムにはタイトルとカードの一覧を表示する |
| F-04 | 各カラムのカード枚数をタイトル横に表示する |

### 3.3 カード

| ID | 要件 |
|----|------|
| F-05 | カードはタイトル（テキスト）を持つ |
| F-06 | カラム内の「+ カードを追加」ボタンからカードを新規作成できる |
| F-07 | カード作成時にタイトルを入力するインラインフォームを表示する |
| F-08 | 空のタイトルではカードを作成できない |
| F-09 | カードを削除できる（削除ボタン or ホバー時に表示） |
| F-10 | カードのタイトルをダブルクリックで編集できる |

### 3.4 カラム間移動

| ID | 要件 |
|----|------|
| F-11 | カード上の「←」「→」ボタンで隣のカラムへ移動できる |
| F-12 | 最左カラムでは「←」ボタンを非表示にする |
| F-13 | 最右カラムでは「→」ボタンを非表示にする |

### 3.5 データ永続化

| ID | 要件 |
|----|------|
| F-14 | ボードの状態（カラム・カード）を localStorage に保存する |
| F-15 | ページリロード後もデータが復元される |

---

## 4. 非機能要件

| ID | 要件 |
|----|------|
| NF-01 | モダンブラウザ（Chrome / Edge / Firefox 最新版）で動作する |
| NF-02 | レスポンシブ対応（モバイル幅でも崩れないレイアウト） |
| NF-03 | 初回表示まで 2 秒以内（ローカル環境） |

---

## 5. 対象外（スコープ外）

- ユーザー認証・アカウント管理
- 複数ボードの管理
- カード詳細（説明文・期日・担当者・ラベル）
- ドラッグ&ドロップによるカード移動
- チームコラボレーション機能
- バックエンド API / データベース連携
