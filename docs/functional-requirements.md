# 機能要件・非機能要件

## 機能要件

### ボード

- アプリ起動時に単一のボードを表示する
- ボードはデフォルトで「To Do」「In Progress」「Done」の3カラムを持つ

### カラム

- 各カラムにはタイトルとカードの一覧を表示する
- 各カラムのカード枚数をタイトル横に表示する

### カード

- カードはタイトル（テキスト）を持つ
- カラム内の「+ カードを追加」ボタンからカードを新規作成できる
- カード作成時にタイトルを入力するインラインフォームを表示する
- 空のタイトルではカードを作成できない（エラーメッセージをインライン表示する）
- カードのタイトルは最大100文字までとする
- カードを削除できる（削除ボタン or ホバー時に表示）
- カード削除時は確認ダイアログを表示し、ユーザーが明示的に承認した場合のみ削除を実行する
- カードのタイトルをダブルクリックで編集できる

### カラム間移動

- カードをドラッグ&ドロップで任意のカラムへ移動できる
- ドラッグ中はドロップ先カラムをハイライト表示し、挿入位置をインジケーターで示す
- カードをドロップした位置に挿入される（カラム末尾への追加も可）

---

## 非機能要件

- モダンブラウザ（Chrome / Edge / Firefox 各最新2バージョン）で動作する
- レスポンシブ対応（モバイル幅 375px 以上でも崩れないレイアウト）
- 初回表示まで 2 秒以内
- ボタン・フォームはキーボード操作（Tab / Enter / Escape）で利用できる
- インタラクティブ要素には適切な `aria-label` を付与し、スクリーンリーダーで操作できる
- API 通信は HTTPS を使用する

---

## 受け入れ基準

- 空のタイトルで追加ボタンを押すと、入力欄の下に「タイトルを入力してください」のエラーメッセージが表示され、カードは作成されない
- タイトルが101文字以上の場合、入力が100文字で止まる（またはエラーを表示する）
- 削除ボタン押下時に「本当に削除しますか？」の確認ダイアログが表示され、キャンセルを選択するとカードが残る
- カードをドラッグ&ドロップで別カラムへ移動でき、移動先の任意の位置に挿入される
- カード追加・移動・削除後にページをリロードしても、操作後の状態が正しく復元される
- 375px幅の画面でカラムが横スクロールまたは縦積みで表示され、要素が画面外にはみ出さない
- Tab キーで全ボタン・入力欄にフォーカスが当たり、Enter で実行、Escape でキャンセルできる
